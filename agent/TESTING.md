# Agent Pack — TESTING

## Быстрые проверки "по умолчанию"

1. Убедиться, что инфраструктура поднята (`make up` или аналог).
2. Проверить health-check Gateway: `GET <GATEWAY_URL>/health`
3. Если задача в рамках конкретной фазы — открыть `phases/phase-X/TESTS.md`.

## Когда тесты обязательны

- Изменения в публичных маршрутах Gateway.
- Изменения в аутентификации (токены, сессии).
- Изменения в real-time соединениях.
- Любые правки, влияющие на изоляцию данных между тенантами.
- Изменения в схеме БД (миграции).

## Принципы тестирования

- Тесты должны быть воспроизводимы в чистом окружении.
- Каждая фаза имеет свой чеклист в `TESTS.md`.
- Проверяй не только "happy path", но и граничные случаи (пустые данные, ошибки авторизации).

## Структура TESTS.md фазы

Каждый `TESTS.md` содержит:
1. Подготовка окружения
2. Чеклист проверок (разбитый по компонентам)
3. Результаты тестирования (заполняется агентом после прохождения)
