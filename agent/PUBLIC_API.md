# Agent Pack — PUBLIC API

## Что считать публичным контрактом

**Публичный контракт** — только то, что доступно через **API Gateway** (единая точка входа).

Всё, что находится "за" Gateway — внутренние детали реализации, не часть контракта.

## Как агенту работать с API

- Задача про интеграцию (Frontend/клиент) — **всегда** начинать с актуальной карты маршрутов.
- Нужна детализация — читать точечно `sources-of-truth/ARCHITECTURE.md`.
- Не предполагать существование эндпоинтов — проверять по документации.

## Принципы

- Все маршруты проходят через Gateway — никаких прямых обращений к сервисам из клиента.
- Аутентификация на уровне Gateway.
- Real-time соединения проксируются через Gateway.
- Версионирование API обязательно при breaking changes.
