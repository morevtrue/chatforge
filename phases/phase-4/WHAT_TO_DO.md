# Phase 4 — AI-чат (диалоги, сообщения, streaming)

**Название фазы:** Основной функционал чата — диалоги и AI-ответы в реальном времени
**Статус:** ✅ Завершена

---

## Цель фазы

End User может создавать диалоги, отправлять сообщения и получать AI-ответы в режиме streaming.
После этой фазы — чат работает как ChatGPT, история сохраняется, лимиты считаются.

---

## Задачи

### 4.1 AI контекст — настройка провайдеров
- [ ] Настроить HTTP-клиент `Req` для OpenAI API (базовый URL, ключ из env)
- [ ] Функция `AI.complete/3` — отправить запрос к OpenAI, вернуть streaming response
- [ ] Поддержка streaming через `Req` + Server-Sent Events от OpenAI
- [ ] Функция `AI.build_messages/2` — собрать массив сообщений (system prompt + история диалога)
- [ ] Логирование использования: сохранить `ai_usage_log` после каждого ответа
- [ ] Конфигурация через env: `AI_PROVIDER`, `AI_MODEL`, `OPENAI_API_KEY`

### 4.2 Chat контекст — Ecto-схемы
- [ ] Схема `ChatForge.Chat.Conversation`
- [ ] Схема `ChatForge.Chat.Message` (role: `user` | `assistant`)
- [ ] Changesets с валидацией

### 4.3 Chat контекст — бизнес-логика диалогов
- [ ] `Chat.create_conversation/2` — создать диалог для end_user в рамках tenant
- [ ] `Chat.list_conversations/2` — список диалогов пользователя (tenant-scoped)
- [ ] `Chat.get_conversation/3` — получить диалог с проверкой принадлежности
- [ ] `Chat.delete_conversation/2` — удалить диалог и все сообщения
- [ ] `Chat.get_messages/2` — получить историю сообщений диалога (с пагинацией)

### 4.4 Chat контекст — бизнес-логика сообщений и лимитов
- [ ] `Chat.send_message/3` — сохранить сообщение пользователя
- [ ] `Chat.check_limit/2` — проверить лимит сообщений (бесплатные + подписка)
- [ ] `Chat.increment_usage/1` — увеличить счётчик `messages_used` у end_user
- [ ] `Chat.save_ai_response/3` — сохранить ответ AI как сообщение
- [ ] Публикация события `message.sent` через PubSub

### 4.5 Phoenix Channel — streaming
- [ ] Создать `ChatForgeWeb.ChatChannel` (`chat:conversation_id`)
- [ ] Событие `send_message` — принять сообщение от клиента
- [ ] Проверить лимит → если исчерпан, отправить событие `limit_reached`
- [ ] Вызвать `AI.complete/3`, стримить чанки через `push(socket, "message_chunk", %{content: chunk})`
- [ ] По завершении стриминга — отправить `message_done` с полным сообщением
- [ ] Обработка ошибок AI API: отправить `message_error` клиенту

### 4.6 Chat — REST API (диалоги)
- [ ] `GET /api/v1/chat/conversations` — список диалогов (tenant-scoped, auth required)
- [ ] `POST /api/v1/chat/conversations` — создать диалог
- [ ] `DELETE /api/v1/chat/conversations/:id` — удалить диалог
- [ ] `GET /api/v1/chat/conversations/:id/messages` — история сообщений (пагинация: `?page=1&per_page=50`)

### 4.7 Chat — лендинг инстанса
- [ ] `GET /api/v1/chat/instance` — публичные данные инстанса (название, цвета, аватар, приветствие, примеры вопросов)
- [ ] Не требует аутентификации

### 4.8 Frontend — лендинг чата
- [ ] Страница `/` (в контексте поддомена) — лендинг с аватаром, приветствием, примерами вопросов
- [ ] Кнопки примеров вопросов — кликабельны, ведут в чат с предзаполненным сообщением
- [ ] Кнопка "Начать чат" — редирект на логин/регистрацию если не авторизован

### 4.9 Frontend — список диалогов
- [ ] Страница `/chat` — список диалогов пользователя
- [ ] Каждый диалог: название (или дата), время последнего сообщения, кнопка удаления
- [ ] Кнопка "Новый диалог"
- [ ] Пустое состояние: иллюстрация + "Начните первый диалог"

### 4.10 Frontend — интерфейс диалога
- [ ] Страница `/chat/:id` — интерфейс диалога
- [ ] Отображение истории сообщений (пузыри: user справа, assistant слева)
- [ ] Аватар AI в сообщениях ассистента
- [ ] Поле ввода сообщения + кнопка отправки (Enter для отправки, Shift+Enter для переноса строки)
- [ ] Streaming: отображать чанки по мере поступления (анимация печатания)
- [ ] Индикатор загрузки пока AI печатает
- [ ] Автоскролл вниз при новых сообщениях
- [ ] Поддержка Markdown в ответах AI (через `react-markdown`)

### 4.11 Frontend — WebSocket подключение
- [ ] Настроить Phoenix JS client (`phoenix` npm package)
- [ ] `src/shared/lib/socket.ts` — создание Socket и Channel
- [ ] Hook `useChat(conversationId)` — подключение к каналу, отправка/получение сообщений
- [ ] Обработка `limit_reached` — показать paywall компонент
- [ ] Обработка `message_error` — показать toast с ошибкой
