# Phases — навигация по фазам разработки ChatForge

Эта папка — "скелет" фазовой документации: **что делать / что сделано / как проверить**.

Важно:
- "Источники истины" находятся в `sources-of-truth/` и имеют приоритет над фазовыми отчётами.
- Фазовые документы фиксируют "актуальную точку входа" для агента.

---

## Структура каждой фазы

```
phase-X/
├── WHAT_TO_DO.md   ← задачи фазы (читает агент перед началом работы)
├── DONE.md         ← что реализовано (агент заполняет по завершении)
└── TESTS.md        ← чеклист проверок (агент выполняет после реализации)
```

## Правила работы с фазами

1. Агент работает с **одной активной фазой** за раз.
2. Перед началом — читает `WHAT_TO_DO.md`.
3. После реализации — заполняет `DONE.md` и проходит `TESTS.md`.
4. Завершённые фазы не редактируются — только для справки.
