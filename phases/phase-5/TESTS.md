- [ ] Таблицы `subscription_plans` и `subscriptions` созданы в БД
- [ ] FK-ограничения и каскадное удаление работают корректно

#### Billing контекст
- [ ] `Billing.create_plan/2` создаёт план с `is_active: true`
- [ ] `Billing.create_plan/2` возвращает ошибку при `price <= 0`
- [ ] `Billing.list_plans/1` возвращает только планы своего тенанта (tenant-изоляция)

- [ ] `Billing.create_subscription/3` корректно вычисляет `expires_at` для monthly и yearly
- [ ] `Billing.get_active_subscription/2` не возвращает истёкшие подписки
- [ ] `ExpireSubscriptionsJob` переводит подписки с `expires_at < NOW()` в `expired`
- [ ] `FakePayment.create_checkout_session/2` возвращает корректный URL

#### API
- [ ] `GET /api/v1/dashboard/plans` — 401 без токена, 200 с токеном Creator-а
- [ ] `POST /api/v1/dashboard/plans` — создаёт план, возвращает 201
- [ ] `PUT /api/v1/dashboard/plans/:id` — обновляет план, возвращает 200
- [ ] `DELETE /api/v1/dashboard/plans/:id` — деактивирует план, возвращает 200
- [ ] `GET /api/v1/chat/plans` — возвращает только активные планы тенанта
- [ ] `POST /api/v1/chat/subscriptions` — возвращает `checkout_url`
- [ ] `GET /api/v1/chat/subscriptions/current` — возвращает активную подписку или null
- [ ] `POST /api/v1/webhooks/payment` — создаёт подписку, возвращает 200

#### Chat лимиты
- [ ] Пользователь с активной подпиской `message_limit: nil` не получает `limit_reached`
- [ ] Пользователь с активной подпиской `message_limit: N` получает `limit_reached` после N сообщений
- [ ] Пользователь без подписки получает `limit_reached` по бесплатному лимиту
