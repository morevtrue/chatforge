# Ролевая система агентов

В этом проекте работа разделена по ролям. Перед началом любой задачи определи её тип и прочитай соответствующий файл.

## Как определить роль

- Задача касается бизнес-логики, API, БД, сервисов → читай `agents/backend.md`
- Задача касается UI, компонентов, состояния, рендеринга → читай `agents/frontend.md`
- Нужно проверить соответствие требованиям, полноту реализации → читай `agents/product.md`
- Нужно найти баги, edge cases, проверить корректность → читай `agents/qa.md`

Если задача затрагивает несколько зон — прочитай все соответствующие файлы.

## Источники истины

Всегда держи в контексте:
- `sources-of-truth/BUSINESS_SPEC.md` — бизнес-требования
- `sources-of-truth/TECH_SPEC.md` — технические решения
- `sources-of-truth/ARCHITECTURE.md` — архитектурные границы
- `rules.md` — правила проекта

## Работа с фазами

Каждая фаза имеет три документа в `phases/phase-N/`:
- `WHAT_TO_DO.md` — задачи фазы (читать перед началом работы)
- `DONE.md` — что реализовано (заполнять по завершении фазы)
- `TESTS.md` — чеклист проверок (заполнять по завершении фазы)

**Правила:**
1. Перед началом фазы — прочитай `WHAT_TO_DO.md` и обнови статус на `🔄 Активная`.
2. После завершения фазы — заполни `DONE.md` (что реализовано, ограничения, что не входило) и `TESTS.md` (результаты проверок).
3. Обнови статус в `WHAT_TO_DO.md` на `✅ Завершена` и в `phases/README.md` обнови таблицу фаз.
4. Завершённые фазы не редактируются — только для справки.
