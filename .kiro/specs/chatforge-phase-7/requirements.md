# Phase 7 — Requirements: Admin-панель платформы

## Обзор

Super Admin получает полноценный инструмент управления платформой ChatForge:
просмотр всех Creator-ов и инстансов, блокировка/разблокировка, мониторинг использования AI API.

---

## Роли

- **Super Admin** — единственная роль с доступом к Admin-панели. Определяется полем `role = "super_admin"` в таблице `users`.
- Все остальные роли (`creator`) получают 403 при попытке доступа к `/api/v1/admin/*`.

---

## Требования

### R1 — Защита Admin-маршрутов

**R1.1** Все маршруты `/api/v1/admin/*` требуют валидный Bearer-токен (через существующий `AuthRequired` plug).

**R1.2** Все маршруты `/api/v1/admin/*` дополнительно проверяют роль `super_admin` через новый plug `RequireSuperAdmin`. При несоответствии — HTTP 403.

**R1.3** Frontend-раздел `/admin` доступен только пользователям с ролью `super_admin`. При попытке доступа с другой ролью — редирект на `/dashboard`.

---

### R2 — Статистика платформы

**R2.1** Super Admin видит сводную статистику: общее количество Creator-ов, активных инстансов, сообщений за всё время, суммарный доход.

**R2.2** Статистика отображается в виде карточек на главной странице `/admin`.

---

### R3 — Управление Creator-ами

**R3.1** Super Admin видит список всех Creator-ов: email, имя, дата регистрации, статус (`active` / `suspended`), количество инстансов.

**R3.2** Список поддерживает поиск по email (частичное совпадение, case-insensitive).

**R3.3** Список поддерживает фильтрацию по статусу (`all` / `active` / `suspended`).

**R3.4** Список поддерживает пагинацию (по 20 записей на страницу).

**R3.5** Super Admin может заблокировать Creator-а (`PUT /api/v1/admin/creators/:id/suspend`). Заблокированный Creator не может войти в систему.

**R3.6** Super Admin может разблокировать Creator-а (`PUT /api/v1/admin/creators/:id/activate`).

**R3.7** Super Admin не может заблокировать самого себя.

**R3.8** При блокировке Creator-а все его инстансы переходят в статус `suspended`.

---

### R4 — Управление инстансами

**R4.1** Super Admin видит список всех Chat Instances: название, поддомен, email Creator-а, статус, дата создания.

**R4.2** Список поддерживает фильтрацию по статусу (`all` / `active` / `suspended`).

**R4.3** Список поддерживает пагинацию (по 20 записей на страницу).

**R4.4** Super Admin может приостановить инстанс (`PUT /api/v1/admin/instances/:id/suspend`). Статус меняется на `suspended`.

**R4.5** Super Admin может восстановить инстанс (`PUT /api/v1/admin/instances/:id/activate`). Статус меняется на `active`.

---

### R5 — Мониторинг использования AI API

**R5.1** Super Admin видит агрегированную статистику использования AI API за период (7д / 30д): суммарные токены (input + output), суммарная стоимость.

**R5.2** Статистика разбита по инстансам: название инстанса, токены, стоимость.

**R5.3** Период задаётся параметром `period=7d|30d`.

---

### R6 — Детали Creator-а

**R6.1** Super Admin может открыть страницу деталей Creator-а: профиль + список его инстансов.

---

## Ограничения

- Создание Super Admin — только через seed/миграцию, не через UI.
- Управление конфигурацией платформы через UI — не входит.
- Просмотр содержимого диалогов пользователей — не входит (приватность).
- Удаление Creator-ов и инстансов — не входит в MVP.
