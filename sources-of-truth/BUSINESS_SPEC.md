# BUSINESS_SPEC — Бизнес-спецификация

## Общая информация

**Название продукта:** ChatForge
**Тип продукта:** B2B / B2C SaaS-платформа
**Текущий статус:** MVP — выделенная инсталляция для одного тенанта.
**Целевая модель:** Multi-tenant SaaS-платформа.

**Краткое описание:**
ChatForge — платформа-конструктор AI-чатботов. Пользователь регистрируется, проходит пошаговый визард настройки и получает готовый AI-чат (аналог ChatGPT) с собственным брендингом, доменом, подписками и монетизацией. Владелец чата может продавать доступ к своему AI-ассистенту конечным пользователям.

---

## Цель продукта

Создать систему, которая позволяет:
- Любому человеку создать собственный AI-чат за несколько минут без навыков программирования.
- Кастомизировать внешний вид, поведение и бренд AI-чата.
- Монетизировать AI-чат через систему подписок для конечных пользователей.

---

## Целевая аудитория

- Предприниматели и инфобизнесмены, желающие продавать доступ к AI-ассистенту.
- Компании, которым нужен брендированный AI-чат для клиентов или сотрудников.
- Фрилансеры и эксперты, создающие AI-помощников в своей нише.
- Образовательные платформы, использующие AI как инструмент обучения.

---

## Два уровня пользователей

Система имеет два принципиально разных уровня:

### Уровень 1: Платформа (ChatForge)
Здесь работают **создатели чатов** — люди, которые регистрируются на платформе и создают свои AI-чаты.

### Уровень 2: Созданные чаты (инстансы)
Каждый созданный чат — это отдельное приложение с собственным URL, брендингом и пользователями. Здесь работают **конечные пользователи** — клиенты создателя чата.

---

## Роли и участники

### Роли платформы (Уровень 1)

| Роль             | Описание                              | Основные возможности                                    |
|------------------|---------------------------------------|---------------------------------------------------------|
| Super Admin      | Администратор всей платформы          | Управление платформой, тенантами, конфигурацией         |
| Creator (Owner)  | Создатель AI-чата                     | Создание, настройка, управление своим чатом и подписками |

### Роли инстанса чата (Уровень 2)

| Роль             | Описание                              | Основные возможности                                    |
|------------------|---------------------------------------|---------------------------------------------------------|
| Chat Owner       | Владелец чата (= Creator с Уровня 1) | Управление настройками, подписками, аналитикой          |
| Subscriber       | Платный пользователь чата             | Полный доступ к AI-чату в рамках тарифа                 |
| Free User        | Бесплатный пользователь              | Ограниченное количество сообщений, затем paywall        |
| Guest            | Неавторизованный посетитель           | Просмотр лендинга чата, регистрация                     |

## Структура тенанта

Каждый созданный AI-чат — это отдельный тенант:

```
Платформа ChatForge
  └── Creator (аккаунт на платформе)
        └── Chat Instance (тенант)
              ├── Настройки (бренд, цвета, аватар, промпт)
              ├── Подписки (тарифные планы)
              ├── Ограничения (лимиты бесплатных сообщений)
              └── Конечные пользователи
                    ├── Subscriber
                    ├── Free User
                    └── Guest
```

> В MVP: один Creator = один Chat Instance.
> В будущем: один Creator может создавать несколько чатов.

## Основные бизнес-сущности

### Chat Instance (AI-чат)
- Что это: отдельный AI-чат с уникальным URL, брендингом и настройками.
- Принципы работы:
  - Создаётся через пошаговый визард.
  - Имеет собственный поддомен (например: `mybot.chatforge.app`).
  - Полностью изолирован от других чатов (данные, пользователи, настройки).
- Ограничения: конечные пользователи одного чата не видят данные другого.

### Builder Wizard (Визард создания)
- Что это: пошаговый конструктор для настройки AI-чата.
- Шаги визарда:
  1. **Цветовая схема** — выбор основных цветов приложения (палитра + ручной выбор).
  2. **Название и валюта** — имя AI-чата (= поддомен) и валюта для подписок.
  3. **Приветствие и аватар** — загрузка аватара AI, текст приветствия, примеры вопросов.
  4. **Подписки и ограничения** — создание тарифных планов, лимит бесплатных сообщений.
- Принципы работы:
  - Можно вернуться на любой шаг и изменить настройки.
  - Превью результата отображается в реальном времени.
  - После завершения — чат сразу доступен по URL.
- Ограничения: визард обязателен для создания чата, нельзя пропустить шаги.

### Subscription Plan (Тарифный план)
- Что это: план подписки, который Creator создаёт для монетизации своего чата.
- Принципы работы:
  - Creator задаёт название, цену, период, лимиты.
  - Конечный пользователь видит paywall после исчерпания бесплатных сообщений.
  - Платёжная система интегрирована на уровне платформы.
- Ограничения: Creator не может менять валюту после создания чата (MVP).

### Conversation (Диалог)
- Что это: отдельный чат-диалог между конечным пользователем и AI.
- Принципы работы:
  - Пользователь может создавать несколько диалогов.
  - История сообщений сохраняется.
  - AI отвечает в режиме streaming.
  - Контекст диалога передаётся AI для связности ответов.
- Ограничения: диалоги изолированы между пользователями.

### Message (Сообщение)
- Что это: единица общения в диалоге (от пользователя или от AI).
- Принципы работы:
  - Сообщения пользователя отправляются через API.
  - Ответ AI стримится в реальном времени.
  - Каждое сообщение учитывается в лимитах тарифа.
- Ограничения: после исчерпания лимита — показывается paywall.

## Пользовательские сценарии (User Flows)

### Flow 1: Creator создаёт AI-чат
1. Регистрация на платформе (email, имя, пароль, телефон, Telegram).
2. Запуск визарда создания.
3. Шаг 1: выбор цветовой схемы.
4. Шаг 2: ввод названия AI и выбор валюты.
5. Шаг 3: загрузка аватара, текст приветствия, примеры вопросов.
6. Шаг 4: настройка подписок и лимитов бесплатных сообщений.
7. Чат создан и доступен по URL.

### Flow 2: Конечный пользователь использует AI-чат
1. Переходит по URL чата.
2. Видит лендинг с приветствием и примерами вопросов.
3. Регистрируется или входит.
4. Создаёт диалог, отправляет сообщения.
5. AI отвечает в режиме streaming.
6. После исчерпания бесплатных сообщений — видит paywall.
7. Оформляет подписку для продолжения.

### Flow 3: Creator управляет чатом
1. Входит в панель управления.
2. Редактирует настройки чата (цвета, аватар, приветствие).
3. Управляет тарифными планами.
4. Просматривает аналитику (количество пользователей, сообщений, подписок).

## Аналитика

Creator (владелец чата) видит:
- Количество зарегистрированных пользователей.
- Количество активных диалогов.
- Общее количество сообщений.
- Количество платных подписчиков.
- Доход по периодам.
- Конверсия из бесплатных в платных.

Super Admin платформы видит:
- Количество созданных чатов.
- Общая активность платформы.
- Использование AI API (токены, стоимость).

---

## Безопасность и доверие

- Строгая изоляция данных между чатами (тенантами).
- Конечные пользователи одного чата не имеют доступа к данным другого.
- Creator видит только аналитику, но не содержимое диалогов пользователей (приватность).
- Гибкая система прав доступа (RBAC).
- Логирование всех действий.
- Защита персональных данных.

---

## Монетизация платформы

### MVP (выделенная инсталляция)
- Бесплатное создание одного чата.
- Платформа берёт комиссию с подписок конечных пользователей.

### Целевая SaaS-модель
- Freemium: один чат бесплатно с ограничениями.
- Pro: несколько чатов, расширенная аналитика, кастомный домен.
- Enterprise: white-label, приоритетная поддержка, SLA.
- Реферальная программа для привлечения новых Creator-ов.
